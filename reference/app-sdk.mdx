`@holaboss/app-sdk` is a thin, type-safe wrapper over the Holaboss REST API. It is generated from the server's OpenAPI surface using [Kubb](https://kubb.dev/), so the types, fetch functions, React Query hooks, and Zod schemas stay in sync with the live contract without any manual maintenance.

## Installation

<CodeGroup>

```bash npm
npm install @holaboss/app-sdk
```

```bash bun
bun add @holaboss/app-sdk
```

</CodeGroup>

React Query hooks are optional. If you use them, add the peer dependency:

<CodeGroup>

```bash npm
npm install @tanstack/react-query
```

```bash bun
bun add @tanstack/react-query
```

</CodeGroup>

<Note>
  Node ≥ 20 is required. Pin an exact version (no `^` range) until the package reaches `1.0.0` — see [Versioning](#versioning) below.
</Note>

## Subpath exports

The package exposes five subpath exports so you only pay for what you import:

| Subpath | Intended for | Contents |
|---|---|---|
| `@holaboss/app-sdk/core` | Non-React consumers, Electron main/preload, Node scripts | Generated fetch functions + `createAppClient` |
| `@holaboss/app-sdk/react` | React UIs | Generated TanStack Query hooks |
| `@holaboss/app-sdk/zod` | Client-side runtime validation | Generated Zod schemas |
| `@holaboss/app-sdk/clients/app` | Advanced: configure the underlying fetch client | Low-level `createClient` + request/response types |
| `@holaboss/app-sdk` | Convenience | Re-exports `core + react + zod` |

## Configure the client

Before calling any generated function or hook, create a configured client instance and share it across your application:

```typescript
import { createAppClient } from "@holaboss/app-sdk/core";

export const appClient = createAppClient({
  baseURL: "https://api.holaboss.ai/api/marketplace",
  credentials: "include",
});

// Pass this object to every generated function / hook
export const appClientOptions = { client: appClient } as const;
```

<ParamField body="baseURL" type="string" required>
  Absolute base URL for the Holaboss product API. Typically `https://api.holaboss.ai/api/marketplace`.
</ParamField>

<ParamField body="credentials" type="RequestCredentials">
  Standard Fetch API credentials mode. Use `"include"` to send cookies for cookie-based auth sessions.
</ParamField>

## Usage patterns

### Plain fetch calls (non-React)

Import from `@holaboss/app-sdk/core` when you need to call the API outside of a React component — for example from Electron's main process, a Node.js script, or a background worker.

```typescript
import { listMarketplaceTemplates } from "@holaboss/app-sdk/core";

const templates = await listMarketplaceTemplates({
  client: appClient,
});
```

### React Query hooks

Import from `@holaboss/app-sdk/react` for components. Every generated hook accepts a `client` option so you can inject your configured instance:

```tsx
import { useListMarketplaceTemplates } from "@holaboss/app-sdk/react";

function TemplatesList() {
  const { data } = useListMarketplaceTemplates({
    client: appClientOptions,
  });
  return <ul>{data?.map((t) => <li key={t.id}>{t.name}</li>)}</ul>;
}
```

### Zod schema validation

Import from `@holaboss/app-sdk/zod` to validate API responses at runtime:

```typescript
import { marketplaceTemplateSchema } from "@holaboss/app-sdk/zod";

const parsed = marketplaceTemplateSchema.parse(rawResponse);
```

### Advanced: low-level fetch client

Import from `@holaboss/app-sdk/clients/app` to access the underlying `createClient` factory and the `RequestConfig` / `ResponseConfig` types if you need to wrap or intercept requests:

```typescript
import { createClient, type RequestConfig } from "@holaboss/app-sdk/clients/app";
```

## Versioning

`@holaboss/app-sdk` follows [Semantic Versioning](https://semver.org/).

| Bump | Meaning |
|---|---|
| **patch** (`0.x.y`) | Internal changes to generated code with no observable API change |
| **minor** (`0.x.0`) | Additive changes: new endpoints, new optional request/response fields |
| **major** | Breaking changes: removed endpoints, removed fields, required field additions, shape changes |

<Warning>
  While on `0.x`, breaking changes may appear on minor bumps per standard semver-for-0.x conventions. They will be called out in release notes. **Pin an exact version** (remove the `^`) until the package reaches `1.0.0`.
</Warning>

## Regeneration

Code is generated from the Holaboss Hono server's OpenAPI spec at `/api/marketplace/openapi.json`. To regenerate with the server running locally:

```bash
bun install
KUBB_APP_OPENAPI_URL=http://127.0.0.1:4000/api/marketplace/openapi.json bun run codegen
```

The `src/generated/` tree is committed to the repository. Codegen is a deliberate human action — it is reviewed in a PR followed by a version bump — and is never run as part of the automated release pipeline.

The Kubb configuration lives in `kubb.config.ts` in the `sdk/app-sdk` package root.
