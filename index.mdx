holaOS is an open-source agent environment built for long-horizon work. Instead of resetting to one-off task execution after every run, holaOS gives agents a structured operating system — runtime, memory, workspace apps, and durable state — so they stay inspectable, continuable, and self-evolving across sessions.

## What is holaOS?

holaOS is the runtime layer that turns a capable LLM into an agent that can work over long horizons. It provides:

- **Durable memory** — agents capture and recall facts across sessions using an embedding-backed memory store
- **Workspace model** — a structured filesystem contract where agents read inputs and write outputs
- **Agent harness** — a stable execution boundary that sandboxes agent runs and streams events back to the desktop
- **MCP integration** — native Model Context Protocol support so agents can call tools exposed by any MCP server
- **Workspace apps** — installable applications with their own lifecycle, ports, and agent-accessible APIs
- **Proactive tasks** — cronjob-driven autonomous runs that let agents act on schedules, not just user input
- **Self-evolving skills** — agents can propose and persist new skill files that improve future behavior

## Get started

<Steps>
  <Step title="Install prerequisites">
    You need `curl`, `bash`, and a macOS, Linux, or WSL machine. Node.js 24 and npm are bootstrapped automatically.
  </Step>
  <Step title="Run the one-line installer">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/holaboss-ai/holaboss-ai/main/scripts/install.sh | bash -s -- --launch
    ```
    This clones the repo, installs dependencies, and launches the Electron desktop app.
  </Step>
  <Step title="Open the desktop workspace">
    Once Electron opens, create a workspace and send your first message to the agent.
  </Step>
  <Step title="Explore and build">
    Read the [learning path](/learning-path) to understand the architecture, then [build your first workspace app](/apps/first-app).
  </Step>
</Steps>
